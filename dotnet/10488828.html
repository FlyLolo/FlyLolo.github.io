本文主要涉及内容：CentOS下Nginx和ASP NET Core 环境的搭建、 多网站多域名的配置部署、SSL证书配置、强制HTTPS访问配置、ASP NET Core反向代理等。

# 1. 概述

本文服务器环境：阿里云ECS   CentOS Linux release 7.6.1810 (Core)

服务器上准备搭建两个网站：

1. 网站服务：www.flylolo.cn     用于页面访问

2. 网站服务：api.flylolo.cn        用于api服务

要求两个网站全部强制https的方式访问。在/home下新建了一个www的目录，接着又分别创建了/home/www/web 和 /home/www/api 两个目录用于存放上述两个网站。

# 2. 配置ASP NET Core环境

通过Putty链接到CentOS服务器，按照该页面上的步骤执行如下命令：

sudo rpm -Uvh https://packages.microsoft.com/config/rhel/7/packages-microsoft-prod.rpm

sudo yum update
    sudo yum install aspnetcore-runtime-2.2

由于只是在此服务器上运行，只安装运行环境就可以了， 没必要安装SDK。   它包含了.NET Core runtime 和the ASP.NET Core runtime。

# 3. 部署ASP NET Core网站

# 1. 网站发布与部署

VS新建一个网站，修改Program文件，

public static IWebHostBuilder CreateWebHostBuilder(string[] args) =>
                WebHost.CreateDefaultBuilder(args).UseUrls("http://*:8081")
                    .UseStartup<Startup>();

这里添加了一句话 UseUrls(<span style="color: #800000;">"http://*:8081")  ，将访问端口指定为8081。  发布网站，并将文件上传到服务器的 /home/www/api 目录下。

测试一下：

cd /home/www/api
    dotnet apidemo.dll

会看到 Now listening on: http:<span style="color: #008000;">//[::]:8081 的提示，网站成功运行。

此时浏览器访问http://服务器ip:8081是无法访问的， 若想测试可以在实例的安全组中配置开放8081端口，建议测试后关闭，因为我们会通过Nginx访问此服务而不是将此端口开放外部访问。

# 2. 配置自动启动

我们不可能每次都通过命令启动服务，而且需要开机自动启动，如下配置一下

再次用nano创建文件：

sudo nano /etc/systemd/system/kestrel-api.service

文件内容如下：

<div class="cnblogs_code_toolbar">[![复制代码](/blogimages/10488828/copycode.gif)]( "复制代码")

[Unit] Description=Example .NET Web API App running on CentOS 7

[Service]
WorkingDirectory=/home/www/api
ExecStart=/usr/bin/dotnet /home/www/api/apidemo.dll
Restart=always
# Restart service after 10 seconds if dotnet service crashes
RestartSec=10
SyslogIdentifier=dotnet-example
User=root
Environment=ASPNETCORE_ENVIRONMENT=Production 

[Install]
WantedBy=multi-user.target
[![复制代码](/blogimages/10488828/copycode.gif)]( "复制代码")

保存并启动服务：

systemctl enable kestrel-api.service
    systemctl start kestrel-api.service

查看是否成功：

systemctl status kestrel-api.service

****会看到绿色的字体提示 Active: active (running) ，若遇到问题，可执行 journalctl -xe 查看错误信息。

至此，我们的dotnet的配置就完成了，下一步用Nginx反向代理访问它。

安装Nginx的依赖的三个包：

安装nginx

1. 安装nginx：#yum install nginx

  1. 启动：systemctl start nginx.service   此时浏览器访问当前IP可以看到写有**Welcome to nginx on Fedora!**的测试页了，这个页面最后一句话： nginx configuration file /etc/nginx/nginx.conf. 提示了配置文件路径。

3. 设为开机自动启动：systemctl enablenginx.service

# 3. SSL免费证书申请

  1. 产品与服务=> 安全（云盾）=> SSL证书。购买证书，选择Symantec的免费型DV SSL。购买后在证书列表中会有一个未签发的证书。

  1. 申请证书：点击申请按钮，会要求填写域名相关信息

注意免费的SSL不能写通配符，例如“*”，即使是二级域名也要单独申请。例如 www.flylolo.cn和 api.flylolo.cn是要申请两个证书的。

特例：若域名只写了flylolo.cn，则可用于flylolo.cn和www.flylolo.cn的访问，而不能用于api.flylolo.cn。

  1. 验证：填写完提交后，会进行域名验证，如果上一步的验证方式选择的是自动DNS验证，则会自动在域名解析处添加一条验证信息，若选择了手工，则需要自己按照提示添加该信息。这条信息仅用于本次验证，验证签发之后，此条自己手动删除即可。

4.提交审核， 验证成功后，点击下面的按钮提交审核，然后等待审核通过。我做这步的时候，最快一分钟左右，刷新一下页面，就发现该证书已经到了【已签发】阶段，这时候就可以下载使用了。

            error_page 404 /404.html;

            }
        }
